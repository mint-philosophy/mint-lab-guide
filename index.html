🏠 What Is This System?

The MINT Lab workspace is a research automation platform built around Minty, an AI research management agent. It runs on a Mac Studio and orchestrates 12 background daemons that automate tasks which would otherwise consume hours of researcher time — fetching, classifying, ingesting, indexing, summarizing, and distributing academic content — while enabling entirely new capabilities like deep multi-agent corpus analysis that simply weren't possible before. Research superpowers, not a replacement for researchers.

Core Idea

Information flows through a pipeline: external sources (Twitter, arXiv, RSS, Substack, Bluesky, email, PhilPapers) are automatically fetched, classified by relevance using Claude Opus, and posted to Slack. Papers are downloaded, converted to markdown via a 6-tier cascade, embedded in a vector database, analyzed across 41 semantic dimensions, and made searchable via a Slack bot. A daily digest email goes out to 28 lab members.

Key Design Principles

  • Delegation-first: The main agent orchestrates; subagents do the work
  • Slack as message bus: All daemons communicate through Slack channels
  • CLI subscriptions, not API: All LLM calls go through claude -p or Codex CLI
  • Graceful degradation: Every stage can fail without stopping the pipeline
  • Single-tier vault: All credentials in macOS login keychain, direct access for all agents and daemons

Adding Papers to the Corpus

One at a time

Upload a PDF directly to #papers in Slack. The ingestion pipeline processes it automatically — you'll see a ✅ reaction and a thread reply when it's done. The paper gets embedded, analyzed across 41 dimensions, summarised, and catalogued in Zotero.

Bulk uploads

Upload PDFs to the local-pdfs folder in Google Drive. They sync down to the Mac Studio and are processed overnight by the batch ingestion daemon (8 concurrent workers). This is the right path for adding dozens of papers at once — don't dump bulk uploads in #papers.

How News Gets Curated

The daily flow: feedme-daily fetches hundreds of items from external sources (Twitter, arXiv, RSS, Bluesky, email, PhilPapers) each morning and classifies them by relevance. The top 20–40% land in a private #feed channel where Seth reviews and thumbs-up the best items. Those are automatically promoted to #firehose by feed-promote. From there, yesterday-in-ai produces a daily digest posted to #mint-community, where lab members discuss AI news and interact with @robo-minty for research questions.

⚡ The Content Pipeline

Here's how information flows from external sources to lab members. Slack's #firehose channel is the automated hub — daemons write classified content here. Three daemons read from it: yesterday-in-ai produces a daily digest for #mint-community, news-pdf-bot finds PDFs, and corpus-ingest feeds the research database.

Key insight: The system forms a funnel. External sources produce hundreds of items daily. feedme-daily classifies them down to ~20-40%. Seth's thumbs-up on #feed curates further. Everything that reaches #firehose gets PDFs attached, ingested into the corpus, and summarized in the daily digest posted to #mint-community.

🔎
slack-index / minty-search
4h index Persistent bot ●

Two-part system. The indexer runs every 4 hours, incrementally embedding all public Slack messages into ChromaDB via Voyage AI voyage-3, with per-channel watermarks and thread replies concatenated into parent documents.

The search bot (@minty-search) is a persistent Socket Mode daemon. When @mentioned, it spawns a full Claude Opus agent with Bash-only tool access (optional Codex gpt-5.3-codex backend configurable via LLM_BACKEND env var). The agent classifies intent, generates 3–5 semantically diverse query variants, executes them via search_dispatch.py, and synthesizes results into flowing prose with inline Slack permalink citations.

Search strategyMulti-query Reciprocal Rank Fusion (RRF, k=60). Agent generates diverse-angle queries, not synonyms. Scores fused across retrieval passes.
Resilience3-tier: Primary agent → Relief officer (fresh context) → Direct search + formatter. Never returns a hard error.
UXHaiku-generated contextual placeholder while Opus works. Whimsical progress messages after 2 min. Catch-up sweep on restart recovers missed queries.
Filtersin:#channel, from:user, days:N. Self-referential detection: "my posts" auto-maps to requesting user.
IndexChromaDB at data/slack_index/, Voyage voyage-3 embeddings
News indexA separate news index (ChromaDB at data/news_index/) indexes #mint-community and #firehose for dedicated news search via @robo-minty's News Search and Catch Me Up modes.
Key filesindex_slack.py, watcher.py, search_dispatch.py, agents/search_agent.md

RAPTOR Summaries

Hierarchical summaries per paper: a document-level overview (~300 words) plus section summaries for Introduction, Methods, Results, Discussion, and Conclusion. These serve dual purposes: fast search index (via document_summary_embedding) and context-efficient paper reading (agents read summaries instead of full text).

Corpus-Ingest Pipeline Detail

Each paper flows through 12 stages in the unified corpus-ingest daemon. The pipeline supports resume from the last completed stage. Models: Codex gpt-5.2 xhigh (analysis + RAPTOR), Voyage voyage-context-3 2048d (embedding).

Deep Review Pipeline Phases

The deep review pipeline runs 4 phases sequentially. The orchestrator coordinates all phases and posts the final output as a Slack canvas.

PhaseModelWhat happens
1. SearchClaude OpusMulti-round search: fast search with semantic query expansion, then up to 2 gap-analysis rounds with targeted queries to fill coverage holes. Uses warm search server with pre-loaded 4GB LanceDB table and Reciprocal Rank Fusion across 49 embedding dimensions.
2. Parallel ReadingCodex GPT-5.2Papers split into two tiers. Top-tier (~3 papers each, 6 parallel readers, xhigh compute) produces deep analytical reports. Summary-tier (batches of 20, medium compute) produces condensed summaries. All readers run in parallel.
3. SynthesisClaude Opus3,000–5,000 word report with Literature Review + Critical Assessment sections. Integrates all reader reports into a unified thematic analysis.
4. QA VerificationClaude OpusCross-references synthesis against reader reports. Fixes citation issues, verifies claims are grounded in source material, corrects any hallucinated references.

Resilience: The old CO hierarchy architecture (autonomous commanding officer dispatching footsoldiers and lieutenants) is archived as a fallback. If the current pipeline fails, the system can fall back to the legacy approach. The system never returns empty-handed if papers were found.

Delegation Protocol

Mandatory rule: "Every tool call you make is context you burn. Subagents have FRESH context. If a task can be described in a prompt, delegate it." The only valid reason to do work yourself is "this literally cannot be described in a prompt."

  • Web search, documentation lookup, file processing, batch operations → always delegate
  • Model policy: Opus for substantive work, Sonnet for retrieval-only, Haiku only for startup data gathering
  • Hooks enforce this: delegation-nudge.py warns after 15 consecutive Read/Grep/Glob calls without a Task delegation
